Texture manager uses a two step mechanism for managing textures. First the textures are registered into a map by their filename. Then the first call to the bind function will cause the initialization of the texture by loading the file and generating the texture data into the graphics memory. This will allow us to call the init function of mglSystem before having a running OpenGL context.
